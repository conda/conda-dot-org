## Changes in conda [25.3.0](https://github.com/conda/conda/releases/tag/25.3.0)/[25.3.1](https://github.com/conda/conda/releases/tag/25.3.1)

To update `conda` to the latest version, run:

```bash
conda install -n base conda=25.3.1
```

**Notable Changes:**
- Add support for Python 3.13.
- Refactor CMD's activation process to use a static shell script instead of a dynamic temporary script. This resolves an issue where using `conda activate` would fail when Windows AppLocker script execution restrictions were enabled.
- `conda 25.3.0` is a deprecation release (see [CEP-9](https://github.com/conda/ceps/blob/main/cep-0009.md) for details) so watch out for deprecated features (previously marked as pending deprecation), many of these deprecated features will be removed in `conda 25.9.0`.

Checkout the full changelog for more: [25.3.0](https://github.com/conda/conda/releases/tag/25.3.0)

## Changes in conda-build [25.3.0](https://github.com/conda/conda-build/releases/tag/25.3.0)/[25.3.1](https://github.com/conda/conda-build/releases/tag/25.3.1)/[25.3.2](https://github.com/conda/conda-build/releases/tag/25.3.2)

To update `conda-build` to the latest version, run:

```bash
conda install -n base conda-build=25.3.2
```

**Notable Changes:**
- Add v0 recipe option `build.python_version_independent` to support building ABI3 for one CPython version and using the package in any later version.
- Add `cdt_arch` and `cdt_name` as used vars when cdt jinja function is used.

Checkout the full changelog for more: [25.3.0](https://github.com/conda/conda-build/releases/tag/25.3.0)

## Changes in conda-libmamba-solver [25.3.0](https://github.com/conda/conda-libmamba-solver/releases/tag/25.3.0)

To update `conda-libmamba-solver` to the latest version, run:

```bash
conda install -n base conda-libmamba-solver=25.3.0
```

Checkout the full changelog for more: [25.3.0](https://github.com/conda/conda-libmamba-solver/releases/tag/25.3.0)
